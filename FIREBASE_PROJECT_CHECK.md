# PC와 모바일이 같은 Firebase 프로젝트 쓰게 하기

## 왜 필요한가요?

- **PC(웹)** 에서 저장한 고객 데이터는 지금 **Firebase 프로젝트 "sos2-49d94"** 에 들어갑니다.
- **모바일(안드로이드/아이폰)** 앱은 아직 "어느 Firebase 프로젝트 쓸지" 설정이 안 되어 있어서, PC와 **다른 저장소**를 보거나 아예 연결이 안 될 수 있습니다.
- 그래서 **PC와 모바일이 같은 저장소(Firebase 프로젝트)를 쓰게** 하려면, 모바일용 설정을 한 번 해줘야 합니다.

---

## 뭘 하면 되나요? (쉽게 말하면)

**"FlutterFire 설정"** 이라는 걸 한 번 돌리면 됩니다.  
이게 하면:

1. PC에서 쓰는 **같은 Firebase 프로젝트(sos2-49d94)** 를 고르고
2. 그 프로젝트에 **안드로이드 앱**, **iOS 앱**을 자동으로 등록하고
3. 프로젝트 안에 **설정 파일**을 알아서 만들어 줍니다.

그래서 **PC와 모바일이 같은 프로젝트(같은 Firestore 데이터)** 를 쓰게 됩니다.

---

## 실제로 할 단계 (복사해서 터미널에 붙여넣기)

### 1단계: 프로젝트 폴더로 이동

- 터미널(또는 VS Code 터미널)을 열고
- 이 앱이 있는 폴더로 이동합니다.  
  예: `cd c:\flutter\project\first_app`

### 2단계: FlutterFire 도구 설치 (한 번만)

아래 명령을 **한 줄씩** 실행합니다.

```
dart pub global activate flutterfire_cli
```

(이미 했으면 다음 단계로 넘어가면 됩니다.)

### 3단계: Firebase에 로그인하고 프로젝트 연결

아래 명령을 실행합니다.

```
flutterfire configure
```

실행하면 대략 이런 식으로 됩니다:

1. **브라우저가 열리면** → Firebase(구글) 계정으로 로그인합니다.
2. **프로젝트 선택**이 나오면 → **"sos2-49d94"** (지금 PC에서 쓰는 프로젝트)를 선택합니다.
3. **플랫폼 선택**이 나오면 → **Android**, **iOS** (모바일 쓸 것만) 선택하고 진행합니다.
4. 끝나면 자동으로:
   - `android/app/google-services.json` 이 생기고
   - `ios/Runner/GoogleService-Info.plist` 가 생기고
   - `lib/firebase_options.dart` 가 PC/모바일 다 쓰는 설정으로 바뀝니다.

### 4단계: 앱 다시 실행

- PC: 웹처럼 그대로 실행
- 모바일: `flutter run` 으로 안드로이드/iOS 실행

이제 **PC에서 저장한 데이터가 모바일에서도 같은 Firebase 프로젝트에서 불러와지고**, **모바일에서 저장한 것도 PC에서 보이게** 됩니다.

---

## 정리

- **말하고 싶은 뜻**: PC와 모바일이 **같은 Firebase 프로젝트**를 쓰게 하려면, **`flutterfire configure` 한 번** 돌려서 모바일용 설정 파일을 만들어 주면 된다.
- **할 일**: 위 1~4단계만 하면 됩니다.
